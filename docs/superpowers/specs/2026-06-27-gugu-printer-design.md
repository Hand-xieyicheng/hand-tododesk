# 咕咕机/便签机打印链接设计

## 背景

在待办事项和备忘录页面增加便签打印能力。用户在桌面端生成一个公开临时网页链接，再用咕咕机、喵喵机等第三方 App 打开该网页并打印当前待办或备忘录内容。

核心约束：

- 打印入口默认不显示，由个人中心配置开启。
- 第三方 App 打开打印链接时不要求登录。
- 链接默认 24 小时有效，后端支持过期和撤销。
- 创建新链接时，后端先删除当前用户已过期的打印链接记录，减少数据库垃圾数据。
- 首版打印当前页面内容，不做历史快照。

## 产品交互

个人中心新增“便签打印”配置区，默认关闭。开启后：

- 待办事项页面顶部操作区显示打印图标。
- 备忘录页面顶部操作区显示打印图标，没有选中备忘录时按钮禁用。

点击打印图标后打开打印配置弹窗。弹窗左侧为配置项，右侧为生成结果区：

- 选择打印模板。
- 选择纸张尺寸或输入自定义宽度。
- 配置字号和边距。
- 选择有效期，默认 24 小时。
- 点击“生成链接”后显示公开打印链接和复制按钮。

待办事项打印范围为当前页面状态：

- 当前标签筛选。
- 是否显示已完成。
- 当前视图类型：列表、四象限或看板。

备忘录打印范围为当前选中的备忘录。生成链接前复用现有保存逻辑，先保存当前备忘录草稿，避免打印旧内容。

## 模板和打印配置

首版提供 4 个预置模板：

1. 清单模板：面向待办，使用勾选框、标题、截止时间和标签等轻量元信息。
2. 便签模板：面向备忘录，保留标题、正文段落、列表、加粗等基础富文本。
3. 省纸模板：小字号、紧凑行距、少装饰，适合长列表或小纸宽。
4. 装饰模板：日期、分隔线、轻量边框，保留小柴记便签感，但以黑白热敏打印效果优先。

纸张尺寸按最终打印宽度建模：

- 预设宽度：57mm、58mm、76mm、80mm。
- 自定义宽度：用户输入后也归一化为最终 `paperWidthMm`。
- 可选最大高度：`maxHeightMm`，首版不强制分页，打印页按连续纸优先。

可配置项：

- `templateId`
- `paperWidthMode`: `preset` 或 `custom`
- `paperWidthMm`
- `fontSizeMode`: `small`、`normal`、`large`、`custom`
- `customFontSizePx`
- `marginMode`: `narrow`、`normal`、`wide`
- `expiresInHours`

生成链接后，配置固化在打印链接记录中。之后用户更改个人默认配置，不影响已生成链接的打印样式。

## 后端数据模型

新增 `PrintShare` 表：

- `id`
- `userId`
- `tokenHash`
- `sourceType`: `tasks` 或 `memo`
- `sourceJson`
- `configJson`
- `expiresAt`
- `revokedAt`
- `lastAccessedAt`
- `createdAt`
- `updatedAt`

明文 token 只在创建接口返回一次。数据库只保存 token hash，避免数据库泄露后可直接访问公开链接。

`sourceJson` 结构：

- 待办：`tagFilter`、`showCompletedTasks`、`viewMode`。
- 备忘录：`memoId`。

`configJson` 保存模板、纸张、字号、边距和有效期对应的最终打印配置。

## API 设计

### `POST /print-shares`

登录态接口。创建公开打印链接。

服务端流程：

1. 校验请求 schema。
2. 删除当前用户过期记录：`DELETE FROM PrintShare WHERE userId = ? AND expiresAt < NOW()`。
3. 校验待办打印范围或备忘录归属。
4. 生成高熵随机 token。
5. 保存 token hash、来源和配置。
6. 返回公开 URL、过期时间和记录 ID。

只清理当前用户的过期记录，不删除其他用户数据。已撤销但未过期的记录首版保留到过期后再由创建流程清理。

### `DELETE /print-shares/:id`

登录态接口。撤销当前用户自己的打印链接，将 `revokedAt` 更新为当前时间。首版弹窗可以不提供管理列表，但接口保留撤销能力。

### `GET /print/:token`

公开 HTML 页面，不要求登录。

服务端流程：

1. hash token 后查询 `PrintShare`。
2. 校验存在、未过期、未撤销。
3. 按 `sourceType` 读取当前用户对应数据。
4. 按 `configJson` 渲染轻量 HTML。
5. 更新 `lastAccessedAt`。

公开打印页不加载主应用 shell、侧边栏、用户邮箱、用户资料等非打印必要信息。

## 前端结构

新增共用打印弹窗组件，例如 `PrintShareDialog`：

- 接收 `sourceType`、`sourceState`、默认配置和打开状态。
- 内部管理模板、纸张、字号、边距和有效期。
- 调用 `api.createPrintShare()`。
- 生成后显示链接和复制按钮。

待办页：

- 打印图标放在 `App.tsx` 的待办顶部操作区，与标签筛选、视图切换和新增按钮同层。
- 创建链接时使用 `App.tsx` 已持有的当前筛选状态、已完成显示状态和视图类型。

备忘录页：

- 复用现有 `saveCurrentMemo()`。
- 通过 `MemoPanel` 现有顶部操作区 portal 放置打印图标，与“当前”“新建”同层。
- 只有存在选中备忘录时允许生成链接，没有选中项时按钮禁用。

个人中心：

- 新增 `printButtonEnabled` 偏好项。
- 默认 `false`。
- 延续现有 `/preferences/theme` 持久化路径：共享 schema、API route、Prisma schema、SQL migration、`ensureIncrementalSchema()` 都需要同步更新。

## 打印页渲染

打印 HTML 由后端输出，目标是第三方 App 内置浏览器可直接打开和打印：

- 使用纯 HTML/CSS。
- 页面宽度按 `paperWidthMm` 设置。
- `@media print` 隐藏非打印状态文案。
- 避免依赖桌面端登录态、前端路由和复杂 JS。
- 备忘录富文本只允许安全的基础标签，复用现有富文本清理思路。

无效链接错误页：

- 链接不存在、已过期、已撤销时显示统一轻量错误页面。
- 不暴露用户名、邮箱、内部 ID 或原始数据是否存在。

## 测试计划

共享包：

- 打印配置 schema 接受四个模板、常见纸宽、自定义宽度、字号和边距。
- 拒绝非法模板、非法尺寸和过长有效期。

API：

- `POST /print-shares` 创建链接成功并返回公开 URL。
- 创建链接前只删除当前用户已过期记录。
- 不删除其他用户过期记录。
- 不删除当前用户未过期记录。
- 备忘录来源必须属于当前用户。
- `DELETE /print-shares/:id` 只能撤销当前用户自己的链接。
- `GET /print/:token` 对有效 token 返回 HTML。
- 过期、撤销、不存在 token 返回错误 HTML。

桌面端：

- 偏好默认关闭，待办和备忘录不显示打印图标。
- 个人中心开启后显示打印图标。
- 待办打印请求包含当前标签筛选、已完成显示状态和视图类型。
- 备忘录生成链接前先保存当前草稿。
- 弹窗生成链接后显示复制按钮。

视觉/打印：

- 四个模板在 57mm、58mm、76mm、80mm 宽度下内容不横向溢出。
- `@media print` 下隐藏复制按钮和状态提示。

## 首版不做

- 打印历史管理页。
- 生成时内容快照。
- 二维码分享。
- 云端定时清理任务。
- 复杂分页和图片压缩策略。

这些能力可以在 `PrintShare` 表和打印页稳定后继续扩展。
